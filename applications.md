# Prompt 应用领域
场景一：根据文本写代码 (Text-to-Code)
痛点： 模型直接给代码，往往逻辑有漏洞，或者由于上下文缺失导致无法运行。应用技术： CoT (先思考后写), Structured Output (结构化输出), Reflexion (自我纠错)。
```
# Role
你是一个资深的 [Python/Java/C++] 工程师，擅长编写健壮、高性能且易于维护的代码。

# Goal
根据用户的需求描述，完成代码编写。

# Constraints
1. **思考过程 (CoT)**: 在写代码前，必须先在 <thinking> 标签内进行逻辑推演。
2. **伪代码**: 先列出伪代码或架构设计。
3. **完整性**: 代码必须包含必要的导入、错误处理 (Try-Catch) 和详尽的注释。
4. **自我反思 (Reflexion)**: 代码写完后，在 <critique> 标签内检查是否有边缘情况 (Edge Cases) 被遗漏，如有则优化。

# Input
[在此处粘贴你的功能需求，例如：写一个Python脚本，遍历文件夹下的PDF，提取文件名中的日期并重命名]

# Response Format
<thinking>
1. 分析需求...
2. 确定关键库...
3. 逻辑步骤...
</thinking>

```python
# 你的代码
```
<critique>

潜在风险分析...

优化建议... 
</critique>
为什么这样更好？**

  * 它强制模型先**Thinking (CoT)**，避免了一上来就写错逻辑。
  * 加入了**Reflexion (反思)**，让模型自己充当 Code Reviewer，能显著减少 Bug。

场景二：深度理解概念 (Deep Concept Understanding)

**痛点：** 问“什么是装饰器”，模型往往给一堆“应用场景”的废话，而不是告诉你“它到底是怎么运行的”以及“语法细节”。
**应用技术：** `Few-Shot` (类比教学), `First Principles` (第一性原理), `Anti-Hallucination` (去废话)。

```
# Role
你是一个崇尚“费曼学习法”的技术专家。我不想要维基百科式的定义，也不要泛泛的应用场景。

# Goal
请向我解释 Workflow 的本质含义和具体用法。

# Instructions
1. **直觉模型 (Analogy)**: 用一个生活中的类比（Few-Shot思想）来解释它的核心机制。
2. **底层原理**: 它是如何工作的？（例如：内存是怎么分配的？数据流是怎么走的？）
3. **最小可行性案例 (MVP)**: 不要写伪代码，给我一段可以直接运行的、最简单的 "Hello World" 级别代码，展示它是如何被**调用**和**执行**的。
4. **常见误区**: 告诉我新手最容易在哪里出错。

# Example (Few-Shot Style)
User: 解释 "Python 装饰器"
AI: 
- **类比**: 就像给一个已经包装好的礼物盒再包一层包装纸，或者给一个人穿上一件钢铁侠战衣。人还是那个人（函数逻辑不变），但有了飞行的能力（增加了功能）。
- **原理**: 本质是高阶函数，函数作为参数传入，返回一个新的函数。
- **MVP**: (展示简单的 @wrapper 代码)

# Input
[在此处输入你想了解的概念，例如：Transformer 中的 Self-Attention 机制]

场景三：论文/长文总结 (Paper Summarization)
痛点： 扔进去一篇论文，模型总结得太笼统，丢失了关键的数据、方法论或创新点。 应用技术： Structured Output (结构化), Chain of Density (密度链 - 逐步浓缩), Prompt Chaining (分步处理)。
```
# Role
你是一个学术界的顶级审稿人 (Reviewer)，此时需要快速评估一篇论文的核心价值。

# Goal
对输入的内容进行深度总结，提取高信噪比的信息。

# Workflow (Prompt Chaining 思想)
请按以下步骤处理输入的文本：

1. **一句话定位**: 这篇文章试图解决什么具体问题？(Problem Statement)
2. **核心方法 (The "How")**: 作者提出了什么新颖的方法/架构？与现有方法有何不同？请具体到技术细节（如具体的算法名称、层数、公式逻辑）。
3. **关键数据**: 提取文中提到的关键指标 (Metrics)，例如准确率提升了多少？速度快了多少？（必须包含数字）。
4. **结构化摘要**:

| 维度 | 内容摘要 |
| :--- | :--- |
| **背景/痛点** | ... |
| **创新点** | ... |
| **实验结论** | ... |
| **局限性** | (模型指出的不足或未来工作) |

# Input
[在此处粘贴论文摘要或全文]
```
